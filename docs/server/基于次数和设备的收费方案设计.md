
# 基于次数和设备的收费方案设计

## 设计原则

1. **从时间控制转向次数控制**：更加精确和公平的计费方式
2. **设备作为核心维度**：多设备使用需要更高套餐
3. **功能模块分离计费**：不同功能使用不同的次数配额
4. **灵活的套餐设计**：满足不同用户群体的需求

---

## 新的收费模型

### 核心变化

| 维度 | 旧方案 | 新方案 | 说明 |
|------|--------|--------|------|
| 养号控制 | 每日时长 | 每日次数 | 按养号启动次数计费 |
| 创作控制 | 月度次数 | 每日次数 | 更精细的创作控制 |
| 导出控制 | 月度次数 | 每日次数 | 更精细的导出控制 |
| 设备控制 | 最大设备数 | 最大设备数 | 保持不变，作为核心维度 |
| 主图生成 | - | 每日次数 | 新增主图生成次数限制 |
| 封面生成 | - | 每日次数 | 新增封面生成次数限制 |

## 次数统计规则

### 1. 养号次数统计
- **统计时机**: 每次成功启动养号时计数+1
- **不计费场景**: 
  - 养号启动失败（如设备连接失败）
  - 养号在1分钟内主动停止
- **重置周期**: 每日 00:00 重置
- **设备限制**: 免费版每天只能选择一个设备进行养号

### 2. 创作次数统计
- **统计时机**: 每次成功创建笔记/PDF转图片时计数+1
- **不计费场景**:
  - 创作保存失败
  - 创作草稿未发布
- **重置周期**: 每日 00:00 重置

### 3. 导出次数统计
- **统计时机**: 每次成功导出内容时计数+1
- **不计费场景**:
  - 导出失败
  - 重复导出同一内容（5分钟内）
- **重置周期**: 每日 00:00 重置

### 4. 主图次数统计
- **统计时机**: 每次成功生成主图时计数+1
- **不计费场景**:
  - 主图生成失败
  - 重新生成同一主图（5分钟内）
- **重置周期**: 每日 00:00 重置

### 5. 封面次数统计
- **统计时机**: 每次成功生成封面时计数+1
- **不计费场景**:
  - 封面生成失败
  - 重新生成同一封面（5分钟内）
- **重置周期**: 每日 00:00 重置

## 数据库设计变更

### 新增表：daily_usage_count（每日使用次数统计表）

```sql
CREATE TABLE daily_usage_count (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id TEXT NOT NULL,
    use_date TEXT NOT NULL,
    yanghao_count INTEGER DEFAULT 0,
    create_count INTEGER DEFAULT 0,
    export_count INTEGER DEFAULT 0,
    main_image_count INTEGER DEFAULT 0,
    cover_image_count INTEGER DEFAULT 0,
    yanghao_device_id TEXT,  -- 记录当天养号的设备ID（免费版限制用）
    update_time TEXT NOT NULL,
    UNIQUE(device_id, use_date)
);
```

### 变更表：user_license（用户授权表）

```sql
-- 新增字段
```

## 迁移策略

### 1. 数据迁移
- 保留原有的时间统计数据（用于参考）
- 新增次数统计表
- 平滑过渡，不影响现有用户

### 2. 用户通知
- 提前通知用户收费模式变更
- 提供过渡期（如1个月）
- 现有用户可选择继续使用旧方案或迁移到新方案

### 3. 兼容性处理
- 服务端同时支持两种验证逻辑
- 客户端根据返回数据自动适配
- 渐进式升级，确保系统稳定

## 优势分析

### 相比旧方案的优势

1. **更精确的计费**: 按实际使用次数计费，更加公平
2. **更好的用户体验**: 用户可以精确控制使用量
3. **更灵活的套餐**: 满足不同层次用户需求
4. **更低的入门门槛**: 免费版提供基础次数，降低试用成本
5. **更高的收入潜力**: 高级版提供完整功能无限制
6. **更完整的功能覆盖**: 新增主图和封面生成的次数控制

### 对用户的好处

- 轻量用户：免费版够用，成本更低
- 中度用户：基础版性价比高
- 重度用户：高级版无任何限制
