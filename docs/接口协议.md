# 激活码API接口协议

本文档定义了所有API接口的请求格式和响应格式。

## 通用说明

### 请求头

所有API请求都必须携带以下请求头：

```
X-API-Key: {你的API_KEY}
Content-Type: application/json
```

### 通用响应格式

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 状态码 |
| message | string | 说明文字 |
| data | object/null | 返回数据 |

---

## 1. POST /auth/verify - 用户端验证激活码

**用途：** 用户在Chrome插件中验证激活码

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| auth_code | string | 是 | 激活码，格式 `{duration}_{20位随机串}` |
| machine_code | string | 是 | 设备唯一识别码（硬件/安装生成） |
| plugin_version | string | 否 | 插件版本号 |

### 响应状态

| status | 说明 |
|--------|------|
| valid | 激活码有效，已绑定当前设备 |
| invalid | 激活码不存在或格式错误 |
| expired | 激活码已过期 |
| used | 激活码已绑定其他设备 |

### 有效响应示例

```json
{
  "status": "valid",
  "message": "激活码验证成功",
  "data": {
    "expiry_date": "2026-03-19T14:05:30.600904",
    "activated_date": "2026-03-16T14:05:30.600894",
    "machine_code": "test-machine-1"
  }
}
```

---

## 2. POST /auth/generate - 管理端生成激活码

**用途：** 后台生成新的激活码

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| duration | integer | 是 | 有效期（天数），支持**任意自定义天数**，比如 1/3/30/999 |
| count | integer | 否 | 生成数量，默认 1，最大 100 |
| package_type | string | 是 | 套餐类型，可自定义（basic/premium等） |

### 激活码格式

`{duration}_{20位字母数字随机字符串}`

示例：`3_2puPQRGFpSDoCUQc3c57`

- 长度适中，方便复制
- 20位随机串碰撞概率极低

### 响应示例

```json
{
  "status": "success",
  "message": "成功生成 2 个激活码",
  "data": {
    "auth_codes": [
      "3_2puPQRGFpSDoCUQc3c57",
      "3_8XErIASWglbbDdz9CfCw"
    ],
    "duration": 3,
    "generate_date": "2026-03-16T14:05:30.347256"
  }
}
```

---

## 3. POST /auth/list - 管理端列出所有激活码

**用途：** 后台查看所有激活码列表

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| offset | integer | 否 | 分页偏移，默认 0 |
| limit | integer | 否 | 每页数量，默认 10 |
| activated_only | boolean | 否 | 只返回已激活的，默认 false |

### 响应示例

```json
{
  "status": "success",
  "message": "获取列表成功",
  "data": {
    "total": 4,
    "offset": 0,
    "limit": 10,
    "list": [
      {
        "auth_code": "3_2puPQRGFpSDoCUQc3c57",
        "duration": 3,
        "package_type": "premium",
        "generate_date": "2026-03-16T14:05:30.347256",
        "activated_date": "2026-03-16T14:05:30.600894",
        "machine_code": "test-machine-1",
        "expiry_date": "2026-03-19T14:05:30.600904"
      }
    ]
  }
}
```

字段说明：
- `activated_date` - 激活时间，`null` 表示未激活
- `machine_code` - 绑定的设备码，`null` 表示未绑定
- `expiry_date` - 过期时间，`null` 表示未激活

---

## 4. POST /auth/delete - 管理端删除激活码

**用途：** 后台删除指定激活码

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| auth_code | string | 是 | 要删除的激活码 |

### 响应示例

```json
{
  "status": "success",
  "message": "激活码 3_2puPQRGFpSDoCUQc3c57 删除成功",
  "data": null
}
```

---

## 5. POST /auth/update - 管理端更新激活码信息

**用途：** 后台更新激活码信息（延期、修改套餐等）

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| auth_code | string | 是 | 要更新的激活码 |
| duration | integer | 否 | 更新有效期（天数），更新后会重新计算过期时间 |
| package_type | string | 否 | 更新套餐类型 |

至少需要提供一个要更新的字段。

### 响应示例

```json
{
  "status": "success",
  "message": "更新成功",
  "data": {
    "auth_code": "3_2puPQRGFpSDoCUQc3c57",
    "duration": 100,
    "package_type": "premium",
    "generate_date": "2026-03-16T14:05:30.347256",
    "activated_date": "2026-03-16T14:05:30.600894",
    "machine_code": "test-machine-1",
    "expiry_date": "2026-06-24T14:05:30.600904"
  }
}
```

## 完整接口列表

| 接口 | 方法 | 用途 | 需要API Key |
|------|------|------|------------|
| `/auth/verify` | POST | 用户端验证激活码 | ✅ |
| `/auth/generate` | POST | 管理端生成新激活码 | ✅ |
| `/auth/list` | POST | 管理端列出所有激活码 | ✅ |
| `/auth/delete` | POST | 管理端删除激活码 | ✅ |
| `/auth/update` | POST | 管理端更新激活码信息 | ✅ |

## 示例curl命令

### 生成激活码

```bash
curl -X POST "https://1259223433-45s5ysqkrg.ap-beijing.tencentscf.com/auth/generate" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"duration": 30, "count": 3, "package_type": "premium"}'
```

### 验证激活码

```bash
curl -X POST "https://1259223433-45s5ysqkrg.ap-beijing.tencentscf.com/auth/verify" \
  -H "Content-Type: application/json" \
  -d '{"auth_code": "30_abcdefghijklmnopqrst", "machine_code": "device-123"}'
```

### 列出所有激活码

```bash
curl -X POST "https://1259223433-45s5ysqkrg.ap-beijing.tencentscf.com/auth/list" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"offset": 0, "limit": 20}'
```

### 删除激活码

```bash
curl -X POST "https://1259223433-45s5ysqkrg.ap-beijing.tencentscf.com/auth/delete" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"auth_code": "30_abcdefghijklmnopqrst"}'
```

### 更新激活码（延期到100天）

```bash
curl -X POST "https://1259223433-45s5ysqkrg.ap-beijing.tencentscf.com/auth/update" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"auth_code": "30_abcdefghijklmnopqrst", "duration": 100}'
```
