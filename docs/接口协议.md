# 激活码接口协议设计

## 1. 接口概述

本接口协议用于管理Chrome插件的激活码和权限，包括验证激活码、生成激活码和权限管理三个主要功能。

## 2. 基础信息

- **API Base URL**: `https://api.example.com/v1`
- **Content-Type**: `application/json`

## 3. 接口详情

### 3.1 验证激活码接口

**接口路径**: `/auth/verify`
**请求方法**: `POST`
**功能**: 验证激活码是否合规、是否过期、是否有其他设备在使用

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| auth_code | string | 是 | 激活码 |
| machine_code | string | 是 | 机器码（设备唯一标识） |
| version | string | 是 | 版本号 |
| auth_type | string | 是 | 认证类型，如plugin、server等 |
| current_expiry_date | string | 否 | 当前过期时间（ISO格式，可选） |

#### 请求示例

```json
{
  "auth_code": "1_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGA==",
  "machine_code": "abc123def456",
  "version": "1.0.0",
  "auth_type": "plugin",
  "current_expiry_date": "2026-03-14T12:00:00Z"
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | string | 验证状态："valid"（有效）、"invalid"（无效）、"expired"（过期）、"used"（已被其他设备使用） |
| message | string | 状态描述 |
| data | object | 验证成功时返回的详细信息 |
| data.expiry_date | string | 过期时间（ISO格式） |
| data.activated_date | string | 激活时间（ISO格式） |
| data.machine_code | string | 绑定的机器码 |

#### 响应示例

**成功响应**:
```json
{
  "status": "valid",
  "message": "激活码验证成功",
  "data": {
    "expiry_date": "2026-12-31T23:59:59Z",
    "activated_date": "2026-03-13T12:00:00Z",
    "machine_code": "abc123def456"
  }
}
```

**失败响应**:
```json
{
  "status": "expired",
  "message": "激活码已过期",
  "data": null
}
```

### 3.2 生成激活码接口

**接口路径**: `/auth/generate`
**请求方法**: `POST`
**功能**: 生成新的激活码，包含生成时间

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| duration | integer | 是 | 有效期天数（1、7、30、365、-1表示永久） |
| count | integer | 否 | 生成激活码的数量，默认为1 |
| package_type | string | 是 | 套餐类型（如：basic、premium） |

#### 请求示例

```json
{
  "duration": 30,
  "count": 5,
  "package_type": "premium"
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | string | 生成状态："success"（成功）、"error"（失败） |
| message | string | 状态描述 |
| data | object | 生成成功时返回的详细信息 |
| data.auth_codes | array | 生成的激活码数组 |
| data.duration | integer | 有效期天数 |
| data.generate_date | string | 生成时间（ISO格式） |

#### 响应示例

**成功响应**:
```json
{
  "status": "success",
  "message": "激活码生成成功",
  "data": {
    "auth_codes": [
      "30_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGA==",
      "30_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGB==",
      "30_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGC==",
      "30_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGD==",
      "30_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGE=="
    ],
    "duration": 30,
    "generate_date": "2026-03-13T12:00:00Z"
  }
}
```

**失败响应**:
```json
{
  "status": "error",
  "message": "生成激活码失败：参数错误",
  "data": null
}
```

### 3.3 权限管理接口

**接口路径**: `/auth/permission`
**请求方法**: `POST`
**功能**: 获取用户的权限信息和功能使用次数限制

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| machine_code | string | 是 | 机器码（设备唯一标识） |
| version | string | 是 | 版本号 |
| auth_type | string | 是 | 认证类型，如plugin、server等 |
| auth_code | string | 否 | 激活码（如果有） |

#### 请求示例

```json
{
  "machine_code": "abc123def456",
  "version": "1.0.0",
  "auth_type": "plugin",
  "auth_code": "1_DEcaW1tMEQMDVhQCXBFVSUwdQ1RWW0cJW1tfUEBXXklSVlFBGA=="
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| status | string | 请求状态："success"（成功）、"error"（失败） |
| message | string | 状态描述 |
| data | object | 权限信息 |
| data.auth_status | string | 认证状态："authenticated"（已认证）、"unauthenticated"（未认证） |
| data.permissions | object | 功能权限信息 |
| data.permissions.prompt_word | object | 提示词管理功能权限 |
| data.permissions.prompt_word.daily_limit | integer | 每天最多使用次数（-1表示不限制） |
| data.permissions.prompt_word.enable_like_filter | boolean | 是否启用点赞数过滤功能 |
| data.permissions.download | object | 下载功能权限 |
| data.permissions.download.daily_limit | integer | 每天最多使用次数（-1表示不限制） |
| data.permissions.search | object | 搜索页面功能权限 |
| data.permissions.search.high_value_notes | object | 显示高价值笔记功能权限 |
| data.permissions.search.high_value_notes.daily_limit | integer | 每天最多使用次数（-1表示不限制） |
| data.permissions.search.keyword_expansion | object | 关键词拓展功能权限 |
| data.permissions.search.keyword_expansion.daily_limit | integer | 每天最多使用次数（-1表示不限制） |

#### 响应示例

**已认证用户响应**:
```json
{
  "status": "success",
  "message": "权限获取成功",
  "data": {
    "auth_status": "authenticated",
    "permissions": {
      "prompt_word": {
        "daily_limit": 50,
        "enable_like_filter": false
      },
      "download": {
        "daily_limit": 20
      },
      "search": {
        "high_value_notes": {
          "daily_limit": 100
        },
        "keyword_expansion": {
          "daily_limit": 50
        }
      }
    }
  }
}
```

**未认证用户响应**:
```json
{
  "status": "success",
  "message": "权限获取成功",
  "data": {
    "auth_status": "unauthenticated",
    "permissions": {
      "prompt_word": {
        "daily_limit": 20,
        "enable_like_filter": true
      },
      "download": {
        "daily_limit": 20
      },
      "search": {
        "high_value_notes": {
          "daily_limit": 30
        },
        "keyword_expansion": {
          "daily_limit": 10
        }
      }
    }
  }
}
```



## 4. 错误码

| 错误码 | 描述 |
|--------|------|
| 400 | 请求参数错误 |
| 401 | 未授权（API Key无效） |
| 403 | 禁止访问（权限不足） |
| 404 | 接口不存在 |
| 500 | 服务器内部错误 |

## 5. 安全性要求

1. **传输安全**: 所有接口必须使用HTTPS
2. **API Key**: 每个请求必须包含有效的API Key
3. **机器码生成**: 机器码应基于设备的唯一标识生成，确保唯一性
4. **激活码加密**: 激活码应使用安全的加密算法生成，包含有效期信息
5. **防滥用**: 实现请求频率限制，防止接口被滥用

## 6. 其他说明

1. 激活码格式：`{duration}_{encrypted_data}`，其中duration为有效期天数
2. 激活时间：首次验证成功时记录，后续验证时使用此时间计算是否过期
3. 设备绑定：一个激活码只能绑定一个机器码，防止多设备使用
4. 激活码生成：生成时应记录生成时间，用于后续管理和审计
5. 数据存储：服务端应存储激活码的详细信息，包括生成时间、激活时间、绑定设备等
6. 过期时间计算：
   - 如果auth_code未使用过，服务端根据auth_code中的有效期周期计算expiry_date
   - 如果auth_code已使用过，服务端验证current_expiry_date是否与存储的过期时间一致
   - 首次激活时，expiry_date = 激活时间 + 有效期周期

## 7. 插件端实现建议

1. 机器码生成：使用Chrome插件的storage API存储一个生成的唯一标识符作为机器码
2. 激活流程：
   - 用户输入激活码
   - 插件生成机器码
   - 调用验证接口验证激活码
   - 验证成功后存储激活信息
   - 定期检查激活状态
3. 错误处理：对接口返回的错误进行适当处理，向用户显示友好的错误信息
4. 重试机制：网络错误时实现适当的重试机制，确保激活流程的可靠性
